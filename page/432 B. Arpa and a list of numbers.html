#432 B. Arpa and a list of numbers

time limit per test: 2 seconds

Arpa has found a list containing numbers. He calls a list bad if and only if it is not empty and gcd (see notes section for more information) of numbers in the list is 1.

Arpa can perform two types of operations:

  • Choose a number and delete it with cost x.
  • Choose a number and increase it by 1 with cost y.

Arpa can apply these operations to as many numbers as he wishes, and he is allowed to apply the second operation arbitrarily many times on the same number.

Help Arpa to find the minimum possible cost to make the list good.

Input

First line contains three integers , and — the number of elements in the list and the integers and .

Second line contains integers — the elements of the list.

Output

Print a single integer: the minimum possible cost to make the list good.

Sample input

 
4 23 17
1 17 17 16

Sample output

 
xxxxxxxxxx
40

Sample input

 
xxxxxxxxxx
10 6 2
100 49 71 73 66 96 8 60 41 63

Sample output

 
xxxxxxxxxx
10

Note

In example, number 1 must be deleted (with cost 23) and number 16 must increased by 1 (with cost 17).

A gcd (greatest common divisor) of a set of numbers is the maximum integer that divides all integers in the set. Read more about gcd here.

    LL cost1,cost2,ans=n*x;
    for(i=2;i<=N+1;++i){
        j=0;

        if(i>k)
            do{
                j+=i;
                cost1+=y*(j*(sumc[j]-sumc[j-k-1])-(suma[j]-suma[j-k-1]));
                cost2+=x*(sumc[j-k-1]-sumc[j-i]);

            do{
                j+=i;
